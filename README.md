# SOC & Blue Team Labs

## Overview

This repository documents my hands-on learning from Security Operations Centre (SOC) and Blue Team focused labs. These labs helped me understand how security analysts monitor, investigate, and respond to threats.

## Labs Covered

- Junior Security Analyst Intro
- SOC L1 Alert Triage
- SOC L1 Alert Reporting
- SOC Metrics and Objectives
- SOC Workbooks and Lookups
- Defensive Security Intro
- Introduction to EDR
- Introduction to SOAR
- Systems as Attack Vectors
- Humans as Attack Vectors

## Skills Demonstrated

- Alert triage
- Incident investigation
- SOC reporting
- Threat detection concepts
- Blue team operations
- Understanding attack vectors
- EDR & SOAR fundamentals
- Security analysis thinking
- Documentation of findings

## What I Learned

Through these labs, I learned:

- how a SOC operates in real environments
- how to analyse alerts and determine severity
- how to document findings clearly
- how attackers exploit systems and humans
- how tools like EDR and SOAR support security teams

## Typical SOC Workflow I Practised

1. Review the alert
2. Analyse logs and context
3. Identify suspicious behaviour
4. Determine severity and impact
5. Document findings
6. Escalate if required

## What I Would Do in a Real SOC

If I encountered a real alert, I would:

- validate the alert
- analyse related logs
- identify affected systems
- collect indicators of compromise (IOCs)
- document findings clearly
- escalate high-risk incidents
- support response actions
